## ✏️ خطوة 2 — التعديل (إن احتجت)

افتح ملف `config.js` بأي محرر نصوص (حتى Notepad). كل فيديو له هذا الشكل:

```javascript
{
  id: 2,
  category: "أخبار / وثائقي",
  title: "خبر مولّد بالذكاء الاصطناعي",
  type: "local",                 // "local" = ملف محلي | "youtube" = من يوتيوب
  youtubeId: "",
  file: "news_fake.mp4",         // اسم الملف داخل مجلد videos/
  groundTruth: "مولّد بالذكاء الاصطناعي",   // الحقيقة المطلقة لهذا الفيديو
  hint: "..."
}
```

لتغيير أي فيديو، فقط عدّل `file` أو `youtubeId` و `groundTruth`.

## 🌐 خطوة 3 — النشر مجاناً (GitHub Pages)

1. أنشئ حساباً مجانياً على **github.com** (إن لم يكن لديك)
2. اضغط **"New repository"** → اسمه مثلاً `blurred-realities`
3. اختر **Public** → اضغط **Create repository**
4. اضغط **"uploading an existing file"** واسحب جميع ملفات هذا المشروع (بما فيها مجلد videos كاملاً)
5. اضغط **Commit changes**
6. اذهب إلى **Settings** → **Pages** (في القائمة الجانبية)
7. تحت **Source** اختر **Branch: main** → اضغط **Save**
8. انتظر دقيقة، ستظهر لك رابط مثل:
   `https://your-username.github.io/blurred-realities/`

هذا هو رابط تجربتك الجاهز للمشاركة مع المشاركين! 🎉

## 🔄 تعديل التجربة بعد النشر

أي وقت تريد تعديل شيء:
1. عدّل الملف المطلوب (عادة `config.js`) مباشرة من واجهة GitHub (اضغط على الملف → أيقونة القلم ✏️)
2. اضغط **Commit changes**
3. التحديث يظهر تلقائياً على رابطك في ثوانٍ

لا حاجة لإعادة النشر أو أي خطوة إضافية.

## 📊 جمع البيانات

كل مشارك يضغط زر **"تحميل بياناتك (JSON)"** في نهاية التجربة، فيُحمَّل ملف JSON يحتوي كل إجاباته. اجمع هذه الملفات من المشاركين (مثلاً عبر طلبهم رفعها في نموذج Google Form بسيط، أو إرسالها بالبريد) ثم حلّلها في Excel أو SPSS أو R.

> **ملاحظة:** هذا تطبيق يعمل بالكامل في المتصفح (client-side) — لا يوجد خادم لتخزين البيانات تلقائياً. كل النتائج تبقى محلية عند المشارك حتى يحمّلها ويرسلها لك. هذا الخيار الأبسط والمجاني تماماً.

## 🆘 مشاكل شائعة

**الفيديو لا يظهر:** تأكد أن اسم الملف في `videos/` مطابق تماماً لما في `config.js`، وأن الامتداد `.mp4`.

**يوتيوب لا يعمل:** بعض الشبكات (مثل شبكات الجامعات) تحجب يوتيوب — جرّب من شبكة أخرى أو حوّل تلك الفيديوهات أيضاً لملفات محلية.

**حجم الملفات كبير جداً:** GitHub يقبل ملفات حتى 100 ميجا بايت لكل ملف. إن كان فيديو أكبر، صغّر حجمه باستخدام أي أداة ضغط فيديو مجانية مثل HandBrake.
